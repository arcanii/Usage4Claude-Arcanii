# U4Claude v1.6.0

**U4Claude learns to remember.** History is now visible ÔÇö both in the popover (24h sparkline under every limit row) and on the desktop (four new widget kinds, including a full dashboard and dedicated sparkline variants). Storage is rebuilt to support this without scaling pain.

## Highlights

### Sparklines in the popover
Every limit row in the menu-bar popover now has a 14 px sparkline strip beneath the icon + percentage + reset countdown. Shows the last 24 hours of utilization for that limit, color-matched to the row's accent. Updates live on every successful fetch.

### Storage rebuild (invisible, but important)
The history file moved from `~/Library/Application Support/<bundle>/usage-history.json` (v1.5.x: full-file rewrite per fetch) to `~/Library/Group Containers/group.com.arcanii.Usage4Claude/usage-history.ndjson` (v1.6.0: O(1) line append per fetch).

- **Why move**: the App Group container is the only place the sandboxed widget extension can read history without an extra write path. Single source of truth across both targets.
- **Why NDJSON**: per-fetch I/O drops from "rewrite a multi-MB file" to "append one ~200-byte line". Compaction (trim to last 10,000 samples Ôëł 7 days) runs once per app launch instead of every minute.
- **Migration**: on first launch of v1.6.0, the old JSON file is read, every sample appended to the new NDJSON file, deduplicated by timestamp, capped, and the legacy file deleted. Idempotent ÔÇö re-runs are no-ops.

## Internal changes

### New files
- `Usage4Claude/Helpers/UsageHistorySample.swift` (shared with widget extension): Codable struct + `UsageHistoryFileStore` enum (append, read, compact, rewrite). Uses ISO8601 with fractional seconds via a custom date strategy so roundtrips are exact (the default `.iso8601` strategy truncates to whole seconds). Decoder accepts non-fractional ISO8601 too, for legacy migration.
- `Usage4Claude/Helpers/UsageHistorySampleBridge.swift` (main app): `UsageHistorySample(from: UsageData)` initializer.
- `Usage4Claude/Helpers/SparklineView.swift` (shared): pure SwiftUI Path-based sparkline with optional gradient fill and "current value" dot. Y axis anchored to 0ÔÇô100.
- `Usage4ClaudeWidget/SharedWidgetTypes.swift`: unified `UsageEntry`, `SnapshotOnlyProvider`, `HistoryProvider`, color/reset-text helpers, demo data.
- `Usage4ClaudeWidget/{LargeDashboard,Sparkline,ExtraLarge}Widget.swift`: the four new widget kinds.
- `Tests/Usage4ClaudeCoreTests/UsageHistoryFileStoreTests.swift`: 14 tests.

### Refactors
- `UsageHistoryStore.swift` Ôćĺ `@MainActor ObservableObject` wrapping the shared file store. `@Published samples` lets the popover sparkline reactively update on each fetch.
- `UnifiedLimitRow` wraps its existing HStack in a VStack and adds the sparkline strip. `UsageDetailView.dynamicHeight` row constant 26 Ôćĺ 48 to make room.
- Existing `Usage4ClaudeWidget.swift` drops its private `UsageEntry` / `UsageTimelineProvider` / `ringColor` / `resetText` in favor of the shared versions. No behavior change.
- `Package.swift` adds `UsageHistorySample.swift` to the `Usage4ClaudeCore` test target.
- `project.pbxproj` widget-target `membershipExceptions` extended with `UsageHistorySample.swift` and `SparklineView.swift`.
