# U4Claude v1.6.3

Two small upstream backports.

## What changed

- **Japanese 24h-format hour suffix now uses the correct kanji `時` (U+6642)** in `formatDateHour`, instead of the Simplified Chinese `时` (U+65F6). Visually similar but typographically wrong on Japanese systems. Splits the previous `langCode.hasPrefix("zh") || langCode.hasPrefix("ja")` branch in `TimeFormatHelper.swift` into separate cases. Backported from upstream commit [`753b6bc`](https://github.com/f-is-h/Usage4Claude/commit/753b6bc).

- **Session Key hint wording generalized** — the old "starts with sk-ant-sid01-" text is obsolete (Claude's current cookies use `sk-ant-sid` without the `01-` suffix). All five locales updated: `settings.auth.session_key_hint` and `welcome.session_key_hint` now read along the lines of *"Paste the full sessionKey cookie value (usually starts with sk-ant-sid)"*. Doc-comment example in `SensitiveDataRedactor.swift` updated to match. Backported from upstream commit [`48bccc9`](https://github.com/f-is-h/Usage4Claude/commit/48bccc9).

## Not backported

- Upstream commit [`4b9716a`](https://github.com/f-is-h/Usage4Claude/commit/4b9716a) ("show detail popover from status bar button") was considered but doesn't apply — it removes the multi-provider transparent-anchor-window code added for Codex's wider popover. Our fork doesn't have Codex, doesn't have the anchor-window code, so there's nothing to remove.
