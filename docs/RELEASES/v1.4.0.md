# U4Claude v1.4.0

Desktop widget. Drag the small or medium U4Claude widget onto your desktop and watch your 5-hour and 7-day rings update in real time, mirrored from the menu bar.

## Highlights

- **Two widget sizes:**
  - **Small** — single ring (5-hour preferred, falls back to 7-day) with the percentage and reset countdown.
  - **Medium** — both 5-hour and 7-day rings side-by-side with their reset countdowns.
- **Reads from an App Group snapshot** the main app writes on every successful fetch. The widget never calls the Claude API itself, so it stays sandboxed with no network entitlement and never has to think about Cloudflare or session expiry.
- **Refreshes immediately** on each main-app fetch via `WidgetCenter.shared.reloadAllTimelines()`. Falls back to a 15-minute background timeline tick if the main app isn't running.

## Adding the widget

1. Run U4Claude (so the snapshot file gets written to the App Group container).
2. Right-click on your desktop → **Edit Widgets…**
3. Search for "Claude Usage" and drag the small or medium variant onto the desktop.

## Internal changes

- New widget extension target `Usage4ClaudeWidgetExtension` (bundle id `com.arcanii.Usage4Claude.Usage4ClaudeWidget`).
- App Group capability `group.com.arcanii.Usage4Claude` shared by both targets.
- New helper types in the main app: `UsageSnapshot` (Codable, App Group-backed) and `UsageSnapshotBridge` (`UsageSnapshot.init(from: UsageData)`).
- `DataRefreshManager.fetchUsage` writes the snapshot and pings WidgetCenter on each successful fetch.
- `scripts/build.sh` switched to **automatic signing** with `-allowProvisioningUpdates`. Manual signing was incompatible with App Groups (every entitled bundle id needs its own provisioning profile, and the script doesn't pre-issue them). Automatic + provisioning-updates lets Xcode generate Mac App Distribution / Developer ID profiles transparently.
