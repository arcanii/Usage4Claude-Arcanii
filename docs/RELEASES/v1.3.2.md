# U4Claude v1.3.2

Bug-fix release. **v1.3.0 and v1.3.1 shipped without working Sparkle.** This release actually wires it up.

## What was broken in v1.3.0/v1.3.1

- `SUFeedURL`, `SUPublicEDKey`, and `SUEnableAutomaticChecks` were never present in the shipped Info.plist. The `INFOPLIST_KEY_<NAME>` build setting prefix only honors Apple-known keys; third-party Sparkle keys were silently dropped from the auto-generated plist. Result: Sparkle had no feed to poll and no key to verify against, so "Check for updates" effectively no-op'd.
- The "Check for updates" menu handler tried to read the AppDelegate via `NSApp.delegate as? AppDelegate`. With SwiftUI's `NSApplicationDelegateAdaptor` wrapping the delegate, that cast didn't reliably succeed — the menu action logged "AppDelegate not available" and silently did nothing.

## What's fixed

- Switched from auto-generated Info.plist to a static `Config/Info.plist` that includes the Sparkle keys directly. `GENERATE_INFOPLIST_FILE = NO`; `INFOPLIST_FILE = Config/Info.plist`. Auto-managed values (CFBundleVersion, MARKETING_VERSION, etc.) still flow through via `$(VAR)` placeholders.
- `AppDelegate` now sets a `static private(set) weak var shared: AppDelegate?` reference in its `init()`. `MenuBarManager.checkForUpdates()` reaches the `SPUStandardUpdaterController` via `AppDelegate.shared` instead of the unreliable `NSApp.delegate` cast.

## Migration

v1.3.0 / v1.3.1 users **cannot get this update via Sparkle** (their Sparkle has no feed URL). They have to either:
1. Install v1.3.2 manually from the GitHub Release (download DMG, drag to Applications), or
2. Use right-click → Check for updates… (no-op on broken builds, but the menu still works on v1.3.2+).

After v1.3.2 is installed, all future updates flow normally through Sparkle.

v1.2.0 users still get notified via their legacy GitHub-API checker — they pick up v1.3.2 directly.
