# U4Claude v1.5.1

Two upstream-v2.6.1 fixes backported. Tiny patch ‚Äî but both close real bugs.

## What changed

- **HTTP/3 disabled on every Claude API request.** `request.assumesHTTP3Capable = false` added to `fetchUsage`, `fetchOrganizations`, and `fetchExtraUsage`. macOS may opportunistically negotiate HTTP/3 (QUIC, UDP) for hosts that advertise it via `Alt-Svc`. That's normally fine, except on networks where a TCP-only system proxy is the gate to the outside ‚Äî UDP either gets blocked outright (request fails) or sneaks past the proxy (corporate-policy violation). Forcing HTTP/2 over TCP keeps both behaviors predictable. Backported from upstream commit [`9feb1fc`](https://github.com/f-is-h/Usage4Claude/commit/9feb1fc).

- **Extra Usage amounts now show cents precision.** Before: `$0/$50` (rounding hides anything under $0.50, looks like a broken counter). After: `$0.45/$50`. The limit stays whole-dollar (it's always a round number you set yourself); the *used* amount and the *remaining* amount both get `%.2f`. Touches the `extra_usage.usage_amount` and `extra_usage.remaining_amount` strings in all 5 locales plus the `formattedCompactAmount` Swift formatter. Backported from upstream commit [`42c7f56`](https://github.com/f-is-h/Usage4Claude/commit/42c7f56) ‚Äî minus the `used_credits: Int? ‚Üí Double?` typing change (deferred until we see actual fractional-cents responses from the API).
