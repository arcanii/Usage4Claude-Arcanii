# U4Claude v1.1.0

First proper distribution release of the Arcanii fork — signed with Developer ID and notarized by Apple, so end users can launch without right-click → Open.

## Highlights

- **Notarized DMG.** Downloads from GitHub Releases now open cleanly on every Mac running macOS 13+. Previous v1.0.0 was ad-hoc signed and tripped Gatekeeper.
- **"Session expired" is no longer disguised as a Cloudflare/captcha block.** When the stored session key goes stale, the popover now surfaces the real reason instead of telling you to "visit claude.ai to confirm you're not a robot".
- **About tab shows the build number** alongside the version (e.g. `Version 1.1.0 (2)`), and the copyright line is updated to 2026.

## Fixes

- `ClaudeAPIService.fetchMainUsage` now inspects 403 response bodies for `permission_error` and returns `.sessionExpired`. Previously every 403 was unconditionally mapped to `.cloudflareBlocked`, hiding expired-session errors behind the captcha-recovery instructions.

## Build & distribution

`scripts/build.sh` was overhauled:

- Fixed all `.app` references after the upstream → `U4Claude` rename (the script targeted `Usage4Claude.app` paths that no longer exist).
- Release config now signs with `Developer ID Application: Matthew Mark` (manual style override) and exports with `method = developer-id`.
- Added an automatic notarize + staple step that submits the DMG to Apple's notary service via the `Usage4Claude-Arcanii-notarize` keychain profile, waits for acceptance, and staples the ticket. The step skips with a help message if no profile is configured, so the script is still usable for unsigned local builds.
- Sets `DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer` to work around `xcode-select` pointing at CommandLineTools.

Future releases ship with `./scripts/build.sh` — bump `MARKETING_VERSION` and `CURRENT_PROJECT_VERSION` in the project, run the script, attach the resulting DMG to a GitHub Release.

## Upgrading

If your menu bar is showing a placeholder ring or a "robot" / "captcha" message, your session has expired. Right-click the menu bar icon → Authentication Settings → re-run the WebLogin flow. (After this release, the message will read "session expired" instead, but you still need to re-login.)
