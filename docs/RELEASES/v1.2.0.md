# U4Claude v1.2.0

Auth-failure UX overhaul, multi-account ergonomics, usage-history export, plus the first round of internal cleanup since the fork was made. All P0/P1/P2 backlog items shipped.

## Highlights

- **Auto-prompt re-login on session expiry.** The app now opens the WebLogin window automatically the first time a fetch fails with `.sessionExpired`. No more manual "right-click → Auth Settings → log in again" detour. Throttled to once per expiry, so the prompt won't re-pop on every 60-second refresh tick.
- **Account-switching keyboard shortcuts.** `⌘1`–`⌘9` now switch between the first nine configured accounts. The shortcut shows next to each account name in the right-click menu so it's discoverable.
- **Usage history + CSV export.** Each successful fetch is now recorded to a rolling history (capped at ~7 days at 1-minute refresh, persisted in Application Support). A new "Export history (CSV)" button in the diagnostics view writes timestamps + per-limit percentages + Extra Usage spend to a file you can open in any spreadsheet.
- **Better Cloudflare vs. session-invalid distinction.** Both `fetchMainUsage` and `fetchExtraUsage` now inspect 403 response bodies for `permission_error` before deciding whether the call hit Cloudflare or just a stale session key. Previously the Extra Usage path silently swallowed expired sessions as "feature not available".
- **Build script overrides via `scripts/build.config`.** Other contributors can now ship Developer-ID–signed DMGs without editing the tracked script — drop a gitignored `scripts/build.config` (template at `scripts/build.config.example`) overriding `DEVELOPMENT_TEAM` and/or `NOTARY_PROFILE`.

## Polish

- About tab simplified — now just "U4Claude" + "Version 1.2.0 (3)" instead of the stacked "Usage4Claude / (Arcanii Mod)" branding.
- Spoofed Chrome user-agent bumped from 131 to 140 to match real Chrome traffic better. Cloudflare's heuristics flag stale UAs; bump again periodically.
- All log messages normalized to English. The codebase had a mix of Chinese (inherited from upstream) and English — `log show --predicate 'subsystem == "com.arcanii.Usage4Claude"'` now reads consistently.

## Internal cleanup

- **`ClaudeAPIService` migrated to `async let`.** The DispatchGroup + shared mutable state in `fetchUsage` is gone — replaced with two parallel `async let`s. Half the lines, no more boolean error tracking, natural failure short-circuiting on the main fetch. The completion-handler public API is preserved so callers don't change.
- **`UserSettings.swift` split into focused extensions.** The 1290-line god-file is now ~1026 lines plus three concern-specific extension files (`+LaunchAtLogin`, `+SmartMode`, `+Accounts`). Same singleton, same public API, just navigable by topic.
- **`SemverCompare` extracted from `UpdateChecker`.** The version-comparison logic that decides whether to prompt for an update is now a pure-function helper, testable without spinning up Bundle.main / GitHub.
- **First round of unit tests via SwiftPM.** Run with `swift test`. 11 tests cover `SemverCompare` parse + ordering edge cases, including the v1.0.0 → v1.1.0 update detection path. `Tests/README.md` documents how to extend with more pure-function coverage.
- **Cloudflare HTML detection moved off substring search.** Previously the Cloudflare-block path was triggered by grepping for `<!DOCTYPE html>` in the response body, which would have misfired on any future API error that happens to embed HTML. Now keys off `Content-Type: text/html`.
- **Daily update check deferred until credentials exist.** `DataRefreshManager.scheduleDailyUpdateCheck()` no longer fires from `init()` — moved to `startRefreshing()` with an idempotency guard. First-launch users on the welcome screen don't trigger a wasted GitHub poll.

## Fixed

- Stale "Request blocked by security system. Please visit claude.ai in browser to confirm you're not a robot" error message that surfaced even when the real cause was `.sessionExpired`. (Continuing the v1.1.0 fix — that release corrected the *message*; this release adds the auto-relogin so users don't have to take action manually.)

## Upgrading

If you're on v1.1.0 the in-app update prompt should detect this release automatically once a v1.2.0 GitHub Release is published (assuming you're inside the 24-hour poll window — quit + relaunch to force-check). On expired sessions, you'll now be taken straight to the WebLogin window instead of seeing an opaque error message.
