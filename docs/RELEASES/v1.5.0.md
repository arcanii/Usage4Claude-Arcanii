# U4Claude v1.5.0

Internal-only refactor + test coverage. **No user-visible changes** — same UI, same data flow, same widget. The release exists so future work on the API layer has a real safety net.

## What changed

- **Response models extracted into their own file.** `Organization`, `UsageResponse`, `ExtraUsageResponse`, `ErrorResponse`, and the in-memory `UsageData` / `ExtraUsageData` storage now live in [Usage4Claude/Helpers/ClaudeAPIResponseModels.swift](../../Usage4Claude/Helpers/ClaudeAPIResponseModels.swift). The file is dependency-free (no `L.*`, no `Logger`, no `UserSettings`) so the SwiftPM test target can cherry-pick it.
- **Formatting moved into a dedicated extension file.** Locale-aware display strings (`formattedResetsInHours`, `formattedCompactResetTime`, `formattedUsageAmount`, `statusColor`, …) are now in [Usage4Claude/Helpers/UsageData+Formatting.swift](../../Usage4Claude/Helpers/UsageData+Formatting.swift) as extensions. Same names, same behavior; just no longer entangled with the wire models.
- **24 new tests for the JSON transforms.** [Tests/Usage4ClaudeCoreTests/UsageResponseTests.swift](../../Tests/Usage4ClaudeCoreTests/UsageResponseTests.swift) covers `toUsageData()` (12 tests including the integer-vs-float utilization split, the `(0, null)` "no data" sentinel for 7-day/Opus/Sonnet, and the fractional-second rounding around the `.645` / `.159` boundaries). [Tests/Usage4ClaudeCoreTests/ExtraUsageResponseTests.swift](../../Tests/Usage4ClaudeCoreTests/ExtraUsageResponseTests.swift) covers `toExtraUsageData()` (12 tests for cents→dollars, currency uppercasing/defaulting, `is_enabled` short-circuit, legacy-field fallback, and the new-fields-win-over-legacy precedence). Test count: **11 → 35**.
- **`fetchOrganizations` migrated to async/await.** Public API is now `func fetchOrganizations(sessionKey: String? = nil) async throws -> [Organization]`. The completion-handler form is gone. All three callsites — `AuthSettingsView.validateAndAddAccount`, `WelcomeView.fetchOrganizationAndCreateAccount`, `WebLoginCoordinator.validateSessionKey` — now `Task { @MainActor in … try await … }`.

## Why bother

The transforms in `toUsageData()` and `toExtraUsageData()` carry non-obvious fallback logic (legacy-field shapes for `ExtraUsageResponse`, the `(utilization=0, resets_at=null)` sentinel for missing rows). Without test coverage, every refactor in this area was a guessing game. Now there's a fast feedback loop:

```bash
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer swift test
```

…runs all 35 tests in under 10ms.

## Internal notes

- `ClaudeAPIService.swift` shrank from 1036 → 504 lines.
- `Package.swift` adds `ClaudeAPIResponseModels.swift` to the `Usage4ClaudeCore` library target's `sources`.
- The widget extension is unaffected — `UsageSnapshot` / `UsageSnapshotBridge` already used `UsageData` only as a stored value type, and the storage layout didn't change.
