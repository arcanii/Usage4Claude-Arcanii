# U4Claude v1.6.2

In-app recovery for stuck widgets — no more dropping to Terminal when chronod's cache wedges.

## What changed

- **New popover menu item: "Reset Widgets"** (right-click the menu bar icon → `…` menu → Reset Widgets, or open the popover and click the same item from the `…`). Two tiers:
  - **Default click** — *medium reset*: writes a fresh App Group snapshot, invalidates WidgetKit's configuration cache, and reloads all timelines. No subprocess, no permission prompts. Fixes most "stale data" / "missing kinds in gallery" / "rendering glitch" issues.
  - **⌥-click (Option-click)** — *hard reset*: medium + restarts `chronod` via `Process` spawning `/usr/bin/killall chronod`. Reliable last-resort recovery for chronod cache corruption (the symptom that hit during the v1.6.1 install — all widgets rendering as opaque black). Triggers a TCC "control other processes" prompt on macOS Sequoia+ the first time.

## Why this exists

The v1.6.1 release surfaced a real-world chronod cache wedge: even though the new appex was installed correctly and contained all 5 widget kinds, the running `chronod` daemon kept serving the previous extension's render output, resulting in opaque black widget tiles. The Terminal recipe `killall chronod` cleared it. v1.6.2 bakes that recovery into the popover menu so it's a click away instead of a shell drop.

The medium tier should handle most rendering glitches without needing the chronod nuke. Use ⌥-click only when the medium tier doesn't recover the widgets.

## Internal

- New `Usage4Claude/Helpers/WidgetReloader.swift` (main-app only). Three static methods: `reloadTimelines()` (soft, unused for now but available), `mediumReset(currentData:)`, `hardReset(currentData:)`. The hard tier uses `Process.run()` against `/usr/bin/killall`; main-app sandbox is off (per `Config/Usage4Claude.entitlements`) so subprocess spawning works without additional entitlements.
- `UsageDetailView` adds a "Reset Widgets" `Button` to the popover's `…` menu, between Web Usage and Quit. Action handler reads `NSApp.currentEvent?.modifierFlags.contains(.option)` and dispatches accordingly.
- New `L.Menu.resetWidgets` + `L.Menu.resetWidgetsHelp` localized strings in all 5 locales (en, ja, ko, zh-Hans, zh-Hant).
